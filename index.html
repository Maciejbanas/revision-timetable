HOW TO REVISE

The Method, Not Just The Topics

Reading notes does almost nothing. Here's what actually works for each subject.

🧬
Biology
AQA · C → A
THE SESSION METHOD
1
Read one subtopic on Save My Exams — don't highlight, don't copy. Just read it properly once and try to understand what's happening.
2
Close the notes. Write everything you remember on a blank piece of paper. Don't look until you're done. What you couldn't write = your gaps.
3
Check against SME. For every gap: make an Anki card — but use the exact mark scheme wording, not your own words. The difference between a B and an A in Bio is usually just vocabulary.
4
Do Save My Exams topic questions. For every wrong answer — even ones you were close on — copy the mark scheme answer word-for-word into an error log.
5
For 6-mark and 9-mark questions: write your answer, then literally tick off each mark scheme point you hit. You'll quickly learn what examiners always want.

✗ "speeds up reactions"✓ "biological catalyst that increases the rate of metabolic reactions without being used up"

Maths
Edexcel · B → A*
THE SESSION METHOD
1
Topic questions ONLY until your teacher confirms all topics are done — no past papers yet. Doing past papers before you've learned everything wastes time and builds bad habits.
2
Do every Save My Exams question in a topic bank. Don't skip. Do them in order. The difficulty ramps up and the later ones are the A/A* questions.
3
For every wrong answer: write the full correct method from the mark scheme — every single algebraic step. Don't just read it. Writing it cements it.
4
Build an error log by topic. Come back to your top wrong question types 2 days later and redo them from scratch. If you can't do them — they stay in the log.
5
Once all topics are done → timed past papers under exam conditions. Mark immediately. Error log every wrong answer. Redo those exact question types the next day.
MARK SCHEME RULES
✗ Skipping algebraic steps✓ Show every line — method marks exist even if your final answer is wrong
✗ Forgetting +c on indefinite integrals✓ Always write +c — it loses a mark every single time it's missing
✗ Not stating positive direction in mechanics✓ "Taking upward as positive" — state it first, stick to it throughout
✗ Classifying stationary points without d²y/dx²✓ "d²y/dx² > 0 therefore minimum" — the classification is a separate mark
✗ Hypothesis test with no context in conclusion✓ "Insufficient evidence at 5% to conclude the mean weight has increased"

Computer Science
OCR · B → A
PAPER 1 METHOD (Computer Systems)
1
Read one topic on SME. Close it. Write every key point from memory. Same retrieval approach as Bio. CS Paper 1 is almost entirely memory — what you can't write, you don't know.
2
Make Anki cards using exact mark scheme wording. CS mark schemes are extremely specific — "DNS translates domain names into IP addresses" not "DNS finds the address." Download actual OCR mark schemes from the OCR website and use their exact phrasing.
3
Do SME topic questions. For every 2-mark question you get wrong — copy the MS answer and note what word or phrase was missing. It's usually one specific technical term.
PAPER 2 METHOD (Algorithms & Programming)
1
You cannot revise Paper 2 by reading. You must write algorithms by hand on paper — because that is exactly what the exam asks you to do.
2
For every algorithm: write the pseudocode from memory, then trace through a small example showing every step. Do this until it's automatic with no notes.
3
Practice OOP by writing classes from scratch — create a scenario, design the class, write the constructor, methods, and a subclass with inheritance. The exam gives you a scenario and asks you to do exactly this.
MARK SCHEME RULES
✗ "finds the address"✓ "DNS translates/resolves domain names into IP addresses"
✗ "the OS manages programs"✓ "the OS manages CPU allocation by scheduling processes so multiple programs can run concurrently"
✗ "stores data"✓ "asymmetric encryption uses a public key to encrypt and a private key to decrypt"
✗ "prevents infinite loops"✓ "a base case must be defined to prevent infinite recursion"
✗ "subclass reuses code"✓ "subclass inherits all attributes and methods of the superclass without redefining them"
ALGORITHMS TO KNOW COLD
Bubble sort — compare adjacent pairs, swap if wrong order, repeat n−1 times. O(n²)
Merge sort — divide in half recursively, merge back in order. O(n log n)
Binary search — check middle, discard half, repeat. Must be sorted. O(log n)
BFS — use a queue, visit level by level, mark visited nodes
DFS — use a stack/recursion, go deep then backtrack, mark visited
Dijkstra's — initialise all distances ∞ except start (0), visit smallest unvisited, update neighbours if shorter path found
